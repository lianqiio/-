# Zero-Trust Vault：基于零信任的个人隐私保险柜

这是一个基于浏览器 Web Crypto API 的前端演示项目，目标是展示“零信任”安全设计在个人隐私数据保护中的应用。项目将秘密内容加密后存放为 Base64 密文，并通过设备指纹、恢复密钥、阅后即焚和审计日志等机制，提升数据的安全性与可控性。

## 项目简介

Zero-Trust Vault 是一个模拟的个人隐私保险柜界面，适合用于学习和演示以下安全思想：

- 使用强加密算法保护敏感信息
- 将加密密钥与设备环境绑定，减少被恶意设备解密的可能
- 提供恢复密钥机制，允许在授权设备之外进行受控恢复
- 通过阅后即焚机制防止明文长期留存
- 提供审计日志与攻击演练入口，展示安全事件可视化

## 核心功能

### 1. 高强度加密
- 使用 AES-GCM 进行对称加密
- 使用 PBKDF2 从密码派生密钥
- 每次加密都会生成随机 Salt 和 IV，增强抗攻击能力
- 加密结果以 Base64 字符串形式输出，便于复制与保存

### 2. 零信任设备绑定
- 通过浏览器 User-Agent、屏幕尺寸和时区生成设备指纹哈希
- 加密时将设备指纹参与密钥派生过程，使数据与特定设备环境绑定
- 如果当前设备未被授权，系统会拒绝普通解密操作

### 3. 恢复密钥机制
- 加密时可以额外设置恢复密钥
- 当设备不在信任列表中时，可以通过恢复密钥进行恢复解密
- 恢复解密成功后，当前设备可被自动加入信任列表

### 4. 阅后即焚
- 解密成功后，明文会在 30 秒后自动清空
- 通过倒计时和进度条提示用户尽快复制或截图

### 5. 审计日志与攻击演示
- 所有操作会实时写入审计日志面板
- 支持模拟攻击场景，展示日志与熔断式安全响应效果

## 运行方式

这是一个纯前端项目，无需安装依赖。

### 方式一：直接打开
- 直接双击打开 [index.html](index.html)
- 或在浏览器中打开项目目录下的 index.html

### 方式二：使用本地静态服务器（推荐）
如果你希望更接近实际使用场景，可以使用任意静态服务器，例如：

```bash
python -m http.server 8000
```

然后访问：

```text
http://localhost:8000/
```

## 使用流程

1. 在“数据封存”区域输入机密内容
2. 设置主控密码
3. （可选）设置恢复密钥
4. 点击“执行高强度封存”
5. 复制生成的密文
6. 在“数据提取”区域输入密文和主控密码
7. 若当前设备未授权，则需要提供恢复密钥
8. 解密后，明文会在 30 秒后自动销毁

## 安全说明

这个项目是一个前端演示与学习示例，主要用于展示零信任安全思想，而不是一个生产级密码管理系统。

### 当前实现特点
- 采用浏览器原生Web Crypto API，密码学逻辑由前端执行
- 数据加密与解密都在本地完成
- 设备指纹和信任列表使用浏览器localStorage保存

### 局限性
- 不提供后端服务、用户注册、云端存储或真正的身份认证
- 生产环境中还需要考虑密钥管理、会话保护、后端审计和多因素认证等问题
